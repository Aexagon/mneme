Mneme Greek: memory v0.2.0

The memory Claude Code keeps between sessions.

Every new chat starts from zero. Mneme loads what past chats learned into each session, then writes new, durable learnings back, so the next chat starts further along instead of starting over.

/mneme:remember save a learning /mneme:recall search it /mneme:review pull the inbox /mneme:loop drive a goal

the shape of it

Two tiers, one engine.

Mneme is a memory today. The same markdown engine extends to a knowledge base over documents you curate. Knowing which is which is the whole map, because the two pull in opposite directions: one stays small so it can load everywhere, the other is free to grow because it never loads uninvited.

in practice

What people do with it

Everything below works on the memory tier that ships today. The pattern is the same each time: tell Claude once, and every future session already knows.

Developersthe people who can install it today
  • Use pnpm, tests are vitest in __tests__, never touch /legacy. Said once, known in every future session on that repo. No more re-explaining the house rules after each /clear.
  • The flaky CI test is a timezone bug. Pin TZ=UTC in setup. Six weeks later it resurfaces and the fix is already there, instead of being re-derived from scratch.
  • Terse answers, run typecheck before declaring done, deploy via make ship. A preference that follows you into every repo, not one project's config file.
  • Commit the project cache into the repo. Now every teammate's Claude inherits the same hard-won context. Onboarding that ships with the code.
Power usersnon-developers who live in Claude Code
  • Nicolette's voice is warm, no jargon. Banned words: synergy, leverage. Every session about that client starts knowing it. Brand rules and standing facts stop getting re-typed.
  • Our packages, pricing, and who's who. Captured once and carried into every chat about the business.
Anyonea different axis: iteration, not memory
  • /mneme:loop "make the suite green" --done "npm test exits 0" Drives one goal to completion in the current chat: plan, act, verify, repeat against a criterion you state, learning each pass and stopping honestly when it is done or stuck.

be honest with the people you send it to

What is real today, and what is not yet

Works now

Anything Mneme can learn from a conversation. Preferences, fixes, conventions, client facts, project context. You work, it remembers, the next chat is sharper.

Not yet

Pointing it at a folder of documents and asking questions that synthesize across them. That is tier 2.

  • Ingest 40 papers, then ask across them
  • Build a companion wiki as you read a book
  • Keep a living analysis current from clippings

If someone installs Mneme expecting this today, they bounce. Say so plainly in the README.

the question a sharp user asks first

Why this, and not the memory that ships in the box

Claude Code already has CLAUDE.md (per-project, hand-maintained) and a built-in memory. So the real deltas have to be specific, or the pitch is hollow.

Plain markdownnot a black box
Notes are files you own: grep them, edit them, version them in git, move them between machines.
Discipline by designso it keeps loading
A relevance gate, dedupe, and pruning keep the index lean. Built-in memory bloats; Mneme has an opinion about staying small.
Global and projecttwo tiers of scope
A global cache that follows you everywhere, plus a per-repo overlay. Built-ins are essentially per-project.
Capture that quarantinesthe distiller
It drafts learnings into an inbox you approve from, never straight into your context.
An engine, not a fileskill bay + loop
A bay of skills that read and write the same cache, and /mneme:loop to drive a task to done.
The knowledge tierthe real reason
The wiki over your own documents is the one thing the built-ins flatly do not do. That is the use case that is only Mneme.
